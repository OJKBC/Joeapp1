# 「ひらがなバトル！」改善相談用 説明文

## このゲームは何か
未就学〜小学校低学年向けの **ひらがな読み取り学習ゲーム**（スマホ・タブレット前提）。
音声でひらがな単語を読み上げ、画面の5択から正しい文字列を選ぶ。正解するとヒーローが
妖怪を攻撃し、HPを削りきると次のレベル（妖怪）へ進む。ポケモンのバトル画面＋
波動拳風エフェクトのような演出。

## 技術構成
- **完全フロントエンドのみ。ビルド不要・依存ライブラリゼロ。** 素のHTML/CSS/JS。
- 音声出力は Web Speech API（`speechSynthesis`、TTS）。**マイクは使わない。**
- 効果音は Web Audio API でコード生成（音声ファイルなし）。
- アニメーションは CSS と Web Animations API。
- ローカルでは `python3 -m http.server` 等の静的サーバーで動かす。
- 対象は日本語ブラウザ（読み上げに日本語音声が必要）。

## ファイル構成（3ファイル）
- `index.html` … 画面のマークアップ＋全スタイル（CSS）。末尾で config.js → game.js を読込。
- `config.js` … **設定と問題データ**（ここだけ編集すれば調整可能）。
  - `CONFIG` … ゲームの挙動設定
  - `LEVELS` … 問題データ（レベル＝妖怪1体）
  - `HEROES` … プレイヤーが選ぶキャラ
  - `YOKAI` … 妖怪の絵文字リスト
- `game.js` … ゲームロジック全部（状態管理・出題・判定・演出）。

## 画面遷移
タイトル → ヒーロー選択（4体から1体） → バトル → ゲームオーバー or 全クリア

## ゲームの流れ（バトル）
1. 妖怪が出現。妖怪はレベルごとに固有HPを持つ。
2. 1問ごとに単語を1つ出題（TTSで読み上げ。文字は既定で**画面に表示しない**＝音だけで当てる）。
3. 選択肢は5択。**正解と同じ文字数の単語を優先的に混ぜる**（字数だけで当てられないようにする狙い）。
4. 正解 → スコア+1、妖怪HP-1、ヒーローがビームで攻撃。HP0でレベルアップ。
5. 不正解 → 既定ではライフ（ハート）-1、妖怪が反撃。正解を光らせて教えたあと次の問題へ。
6. ライフが尽きるとゲームオーバー。最終レベルの妖怪を倒すと全クリア。

## CONFIG（調整できる設定）
- `startHearts: 5` … ライフの数
- `yokaiHp: 3` … 妖怪を倒すのに必要な正解数（LEVELS側で個別指定があればそちら優先）
- `choiceCount: 5` … 選択肢の数
- `penaltyMode: 'heart'` … 不正解の罰。`'heart'`（ライフ-1）か `'score'`（スコア-1）
- `speakPrompt: true` … 読み上げのオン/オフ
- `showPromptText: false` … 問題文字を画面に出すか（false=音だけで当てる）
- `rate: 0.65` … 読み上げ速度（小さいほどゆっくり）
- `sound: true` … 効果音のオン/オフ

## 問題データ（LEVELS）の構造
レベルは難易度順に並んでおり、文字の種類で段階化している：
1. 2もじ（あ〜ん）
2. 3もじ（あ〜ん）
3. てんてん（濁点 が・ば 等）
4. まる（半濁点 ぱぴぷ）
5. ちいさい っ（促音）
6. のばすおと ー（長音）
7. ちいさい ゃゅょ（拗音）

各レベルは `{ name, hp, items:[{t:'よみ', p:'絵文字'}, ...] }`。
`t` が読み（＝正解文字列）、`p` がヒントの絵文字。

## 現状わかっている特徴・気になる点（改善のヒント）
- スコアやレベルの**進捗は保存されない**（リロードで最初から）。
- 難易度カーブが固定。子どもの正答率に応じた調整はない。
- 出題は各レベル内シャッフルのみ。**間違えた問題を重点的に出す等の学習最適化はなし。**
- 不正解時もスコアは減るが、**ヒント（絵文字）が常に正解を示してしまう**ので、
  音を聞かなくても絵で当てられる場合がある（学習意図と矛盾する可能性）。
- 文字そのものの認識（1文字ずつ）ではなく単語単位なので、純粋なひらがな学習というより
  単語読みに近い。
- アクセシビリティ（色覚・読み上げ依存）や、日本語TTS音声が無い端末での挙動が未対策。
- 保護者向けの設定UIはなく、設定変更はコード編集が必要。

## ChatGPTへの依頼（このあとに付ける質問例）
上記をふまえて、次の観点で改善案を出してください：
1. **学習効果**を高める仕組み（間違い問題の再出題、1文字認識モード、難易度自動調整 など）
2. **モチベーション設計**（進捗保存、ごほうび、コレクション要素 など）
3. **UX/演出**で子どもが続けたくなる工夫
4. コードを大きく作り替えずに**少ない変更で効く改善**を優先度つきで
それぞれ「狙い・具体的な実装方針・難易度」を添えてください。
